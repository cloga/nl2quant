## Case 1 - 创业板ETF期权卖方策略

### 策略概述
通过卖出期权收取权利金，实现收益增强，避免直接买入ETF的机会成本。

### 初始状态
- **不直接买入**创业板ETF（159915.SZ 或 159977.SZ）
- **卖出10张深度虚值认沽期权**（OTM Put）
  - 虚值度：10%（行权价 = 当前价 × 0.90）
  - 合约规模：10张 × 100股/张 = 1000股对应量
  - 到期时间：当月合约（距离到期约30天）

### 情况1：标的上涨（认沽期权未被行权）

#### 1.1 正常持有到期
- 标的价格 > 行权价 → 期权作废，收取全部权利金
- 下月继续卖出10张深度虚值（10%）认沽期权

#### 1.2 动态移仓（锁定利润）
**触发条件**：同时满足以下两个条件
1. 距离到期还剩 ≤15天
2. 标的价格上涨（当前价 > 开仓时价格）

**操作**：
- 平掉当前10%虚值认沽期权（买入平仓）
- 移仓至下月5%虚值认沽期权（卖出开仓）
- 目的：降低虚值度，提高被行权概率，同时锁定当前利润

### 情况2：标的下跌（认沽期权被行权）

#### 2.1 首次被行权
- 持有到行权日，以行权价接入1000股ETF
- 接入成本 = 开仓时价格 × 0.90（10%虚值）
- 实际成本 = 行权价 - 已收权利金

#### 2.2 持有标的后的策略（双卖策略）
**立即执行**：以**持仓成本价**为基准（而非当前市价）

同时卖出两个期权：
1. **虚值认购期权**（OTM Call）
   - 行权价 = 持仓成本 × 1.10（成本价上方10%）
   - 10张合约
   
2. **虚值认沽期权**（OTM Put）
   - 行权价 = 持仓成本 × 0.90（成本价下方10%）
   - 10张合约

**双卖目的**：
- 认购收权利金，若被行权则以盈利10%卖出标的
- 认沽收权利金，若被行权则继续补仓摊低成本

#### 2.3 连续被行权（追加持仓）
- 如果认沽再次被行权 → 追加1000股，持仓变为2000股
- 重新计算平均持仓成本
- 继续以新的成本价为基准，双卖认购+认沽（各10张）
- **资金管理**：准备多份保证金以应对连续行权

### 情况3：持有标的后，认购被行权

#### 3.1 盈利出场
- 卖出的认购被行权 → 标的价格涨至成本价+10%
- 以行权价卖出1000股，锁定10%利润（不含权利金收益）
- 如果还有剩余持仓，继续执行双卖策略
- 如果全部卖出，回到**初始状态**，重新卖出10张虚值认沽

### 关键参数设定

| 参数 | 数值 | 说明 |
|------|------|------|
| 初始合约张数 | 10张 | 每张100股，对应1000股ETF |
| 虚值度（认沽） | 10% | 行权价 = 现价 × 0.90 |
| 虚值度（认购） | 10% | 行权价 = 成本价 × 1.10 |
| 移仓时间阈值 | 15天 | 距离到期≤15天且上涨时移仓 |
| 移仓后虚值度 | 5% | 移仓至平值附近，提高行权概率 |
| 权利金估算 | 1%-2% | 月度，10%虚值认沽的市场价格 |

### 资金管理要求

根据历史数据分析（2010-2025年创业板指数）：
- **10%虚值认沽**：13次被行权，其中1次连续3个月行权
- **最差情况**：连续3个月被行权，需持有3倍仓位
- **建议准备资金**：
  - 最低3份保证金（应对连续3次行权）
  - 理想5份保证金（留有安全边际）

**示例**（假设ETF价格2元）：
- 单次行权金额：1000股 × 2元 × 0.90 = 1.8万元
- 建议准备资金：1.8万 × 5 = 9万元

1. **收益指标**
   - 总收益率（权利金+价差收益）
   - 年化收益率
   - 夏普比率

2. **风险指标**
   - 最大回撤
   - 最大连续被行权次数
   - 最大资金占用倍数

3. **胜率统计**
   - 月度胜率（收取权利金的月份占比）
   - 被行权频率
   - 双卖策略盈利频率

4. **与基准对比**
   - vs 直接持有创业板ETF
   - vs 纯卖认沽策略（不含双卖）
   - vs 创业板指数

**发现**：

### 优化策略对比

| 策略 | 终值 | 总收益 | CAGR | 最大回撤 | 周期数 | 胜率 | 平均周期 | vs 基准 |
|------|------|--------|------|----------|--------|------|----------|---------|
| **Dynamic DCA（估值分位）** | 2,529,205 | 152.92% | **6.00%** | -35.58% | 21 | 100% | 270天 | **-0.96%** |
| Trailing TP (8%触发+3%回撤) | 2,314,597 | 131.46% | 5.41% | -34.47% | 17 | 100% | 335天 | -1.54% |
| Trailing TP + Dynamic DCA | 2,003,999 | 100.40% | 4.46% | -35.58% | 15 | 100% | 379天 | -2.49% |

**关键发现**：

#### 1. Dynamic DCA（估值分位动态定投）是最优单一优化 ⭐
- **机制**：基于 252 日滚动窗口计算价格分位
  - 价格 < 30% 分位（低估）→ 加倍定投（2W/天）
  - 价格 > 70% 分位（高估）→ 减半定投（0.5W/天）
  - 30%-70% 分位（中性）→ 正常定投（1W/天）
- **效果**：CAGR 6.00%，**仅落后基准 0.96pct**，是所有定投策略中最接近基准的方案
- **优势**：在低估区域快速建仓，高估区域控制成本，显著提升资金效率

#### 2. Trailing TP（移动止盈）单独使用效果一般
- **机制**：浮盈达到 8% 后启动跟踪，回撤 3% 时卖出
- **效果**：CAGR 5.41%，周期数大幅减少（17 个），平均持仓延长至 335 天
- **问题**：8% 触发 + 3% 回撤可能过于保守，错过部分小波段
- **建议**：可测试更激进参数，如 10% 触发 + 4% 回撤

#### 3. 两种机制叠加适得其反
- **Dynamic DCA + Trailing TP** 组合 CAGR 仅 4.46%，不如单独使用
- **原因**：动态定投已优化资金节奏，再加移动止盈导致策略过于保守
- **结论**：复杂策略需要独立调优参数，不能简单叠加

## 核心洞察

### 1. 分红再投资的巨大影响
- 从价格指数（000922.SH）切换到全收益指数（h00922.CSI）后：
  - 基准 CAGR 从 2.58% 跃升至 6.95%（提升 170%）
  - 定投策略从"跑赢基准 +1.02pct"变为"落后基准 -1.41pct"
- **结论**：在长期上涨 + 高分红再投场景下，一次性买入占优；定投策略更适合震荡或下跌市

### 2. 回撤控制是定投核心优势
- 所有定投策略 MDD 在 -17.38% 至 -35.58%，远优于基准 -45.66%
- 即使 CAGR 略低，对于风险厌恶型投资者仍有价值
- 特别适合退休金账户、教育金等不能承受大幅回撤的资金

### 3. 简单往往更有效
- 最优策略（Dynamic DCA）只用了单一优化维度（估值分位）
- 复杂组合（Trailing TP + Dynamic DCA）反而表现更差
- 建议：先测试单维度优化，验证有效后再考虑组合

```python
# 价格动量 (20%)
price_score = 50 + price_change % * 100

# 波动率 (20%)
volatility_30d = 30日收益率标准差 * sqrt(252)
vol_score = 100 - volatility_30d * 100

# 趋势强度 (30%)
trend = MA20 / MA60 - 1
trend_score = 50 + trend * 200

# 估值位置 (30%)
value_score = 50 - (current_price - MA60) / MA60 * 100

# 综合评分
total_score = price_score * 0.2 + vol_score * 0.2 + trend_score * 0.3 + value_score * 0.3
```

## Case 6 - 网格交易策略（50W基础 + 50W网格）

### 策略概述
将100W资金分为两部分：
- **基础仓位**（50W）：长期持有中证红利全收益指数，享受分红再投
- **网格仓位**（50W）：在基础仓位周围按2%间距设置买卖网格，通过低买高卖增强收益

### 核心参数
- 初始资金：1,000,000 元
- 基础持仓：500,000 元（50%）
- 网格资金：500,000 元（50%）
- 网格间距：2%（买入-2%，卖出+2%）
- 单位网格：50,000 元
- 数据源：h00922.CSI（中证红利全收益指数）
- 回测期间：2018-01-02 至 2025-12-05（1922 交易日，7.9 年）

### 基础网格策略回测结果

#### 净值对比
| 策略 | 终值 | 总收益 | CAGR | 最大回撤 | 交易数 | vs 基准 |
|------|------|--------|------|----------|--------|---------|
| **基础仓位（50W B&H）** | 838,097 | 67.62% | 6.73% | -27.08% | 0 | - |
| **网格仓位（50W 独立网格）** | 618,641 | 23.73% | 2.72% | -24.29% | 192 | -4.01% |
| **合并仓位（100W B&H）** | 1,676,194 | 67.62% | 6.73% | -27.08% | 0 | 基准 |
| **基础+网格组合** | 1,456,738 | 45.67% | 4.86% | -21.67% | 192 | **-1.87%** |

#### 关键发现
1. **网格策略单独表现欠佳**
   - 网格仓位 CAGR 仅 2.72%，相对基准跑赢不足 1pct
   - 交易192笔，但无法充分捕捉上升趋势
   
2. **基础+网格组合优化回撤**
   - 最大回撤改善：-27.08% → -21.67%（改善 5.41pct）
   - 但 CAGR 下降 1.87pct，权衡收益换安全性
   
3. **为什么网格在上升趋势中失效**
   - 2018-2025 中证红利呈强上升趋势（7.9 年 CAGR 6.73%）
   - 网格策略设计用于箱体波动，高卖后常被反向涨停踏空
   - 止盈卖出占利润20%，但反弹后无法快速追回

---

### 趋势过滤优化版本

#### 改进逻辑
使用 MA120（120日移动平均）作为趋势过滤器，**修改卖出逻辑**：
- **始终买入**：在设定网格价格下坚决买入（建仓阶段）
- **择时卖出**：仅在 price > MA120（上升趋势）时卖出已有头寸
- **下跌期间**：price < MA120 时暂停卖出，继续累积 → 低成本建仓

#### 趋势过滤回测结果

| 指标 | 无过滤 | MA120过滤 | 改善 |
|------|--------|----------|------|
| **终值** | 1,427,239 | 1,413,310 | -13,929 |
| **CAGR** | 4.59% | 4.46% | **-0.13%** |
| **最大回撤** | -17.79% | -19.36% | **-1.57%** ❌ |
| **交易总数** | 96 | 66 | -30.2% ✅ |
| **vs 基准** | -2.14% | -2.27% | -0.13% |
| **上升天数占比** | 63.5% | 63.5% | - |

#### 趋势过滤诊断

**问题1：反弹策略失效**
- 设计假设：下跌期少卖→低位积累→反弹时获利多倍
- 实际情况：2018-2025 整体上升，很少出现长期下跌 < MA120
- 结果：减少了30%的交易，但错过了2%上升波段的快速平仓机会

**问题2：交易减少反而伤害收益**
- 更少的卖出 → 更少的止盈锁定 → 后期回撤时承担更多浮亏
- 无过滤版通过频繁小额止盈保护了收益稳定性

**问题3：MA120 粘性不够**
- MA120 在上升趋势中粘着价格不放，过滤效果有限
- 趋势反向时反应缓慢（延迟20-40天）

#### 结论
**MA120 趋势过滤不适合此策略**。网格策略在趋势市本身效率不高，盲目加过滤只会雪上加霜。

### 后续优化建议

#### 高优先级（待验证）
1. **动态网格间距**
   - 上升趋势：扩大间距至 3%-5%（避免频繁止盈）
   - 下跌趋势：缩小间距至 1%-1.5%（高频获利）
   - 实现方式：基于 ATR（真实波幅）动态调整

2. **对冲组合**
   - 基础仓 50W：做多中证红利
   - 网格仓 30W：做多
   - 对冲仓 20W：做空或布局看跌期权 → 风险对冲

3. **趋势识别 + 网格切换**
   - 非必需指标，而是模式识别
   - 箱体检测（20日高-低 < 前期 30%）→ 激活网格
   - 突破检测（突破 MA60 + 收盘价 > 前20日最高）→ 关闭网格

4. **资金管理优化**
   - 不是固定 50W 网格，而是浮动
   - 基础仓浮盈达 20% 时，转移部分利润到网格 → 良性循环

#### 中优先级（需要真实数据验证）
5. **多品种网格组合**
   - 同时在债券 ETF（511260）运行网格 → 低相关性
   - 多品种间配置优化

6. **机器学习择时**
   - 用 LSTM 预测未来 5 日 ATR 和趋势
   - 预测高波动期激活网格，预测低波动期关闭网格

#### 低优先级（理论探索）
7. **期权对冲**
   - 卖出虚值认购期权（防止踏空），收权利金补偿
   - 实质：将"做多+网格"变为"做多+卖认购"

8. **时间加权网格**
   - 月初 1-10 日：宽松网格（3% 间距）
   - 月中 10-20 日：正常网格（2% 间距）
   - 月末 20-30 日：激进网格（1% 间距）
   - 适应月度资金面变化
